# Logo rendering on EVE: skeleton-generated PNG decoded on-chip

## Context

The dash needed a galloping-pony logo with no design asset available. Two
problems in one: generating acceptable line art programmatically, and getting
it onto the panel through a 12 KB-class embedded budget.

## Guidance

**Generating the silhouette (tools/pony.py):** naive smoothing of one closed
outline blobs away thin structures — the first attempt produced a fox, not a
horse, because spline interpolation over sparse points erases legs and ears.
The working approach is skeleton-based: a spline-smoothed polygon for the
torso only, plus *tapered strokes* for neck, head, legs, tail, and mane, each
rendered by stamping circles of interpolated radius along a joint path
(tools/pony.py:26-34). Details that make it read as the intended animal:
joint bends in the limbs, slightly bulged stroke ends for hooves, and overlap
between the mane/limb strokes and the torso so no seams show. Render 4x
supersampled, downscale with Lanczos, and iterate by *looking at the preview*
— the shape coordinates are hand-tuned against rendered output, not authored
blind.

**Getting it on screen:** export a white-on-transparent PNG, embed it as a
flash array (MustangDash/pony_png.h, generated by the same script pipeline),
and let the BT817 decode it on-chip — no host-side pixel conversion:

```c
/* MustangDash/MustangDash.ino:102 -- PNG with alpha decodes to ARGB4 in RAM_G */
EVE_cmd_loadimage(MEM_LOGO, EVE_OPT_NODL, pony_png, sizeof(pony_png));
EVE_execute_cmd();
...
/* MustangDash/MustangDash.ino:143 -- draw it tinted (white source * COLOR_RGB) */
EVE_cmd_setbitmap(MEM_LOGO, EVE_ARGB4, PONY_PNG_WIDTH, PONY_PNG_HEIGHT);
```

A white-source PNG is tintable at draw time with `EVE_color_rgb()`, so one
asset serves any color scheme.

**Raw display-list words:** the library ships convenience macros like
`VERTEX2F`/`LINE_WIDTH` commented out (libraries/FT800-FT813/src/EVE.h:1021),
so define locally what you need (MustangDash/MustangDash.ino:39) and remember
`VERTEX2F` coordinates are in 1/16-pixel units — multiply pixel positions by 16.

## Why This Matters

On-chip PNG decode keeps the asset small (12.5 KB PNG vs ~280 KB raw ARGB4
for 480x300), keeps the sketch free of image-conversion tooling, and makes
the logo editable: rerun `tools/pony.py` (WSL python3 + Pillow on this
workstation) and rebuild. The skeleton generator is reusable for any
silhouette; the blob-vs-skeleton lesson generalizes to all programmatic line
art.

## When to Apply

- Any static artwork on EVE panels: prefer PNG + `CMD_LOADIMAGE` over
  precomputed raw bitmap arrays until profiling says otherwise.
- Any programmatic silhouette: model appendages as tapered strokes over a
  skeleton, never as one smoothed outline.
- Any direct display-list composition with this library: expect to define the
  `VERTEX2F`-class macros yourself, in 1/16-pixel units.

## Examples

The committed pipeline: `tools/pony.py` (generator, with the skeleton tables
at tools/pony.py:37-57) -> `MustangDash/pony_png.h` (12,566-byte embedded
PNG) -> `draw_dash()` in MustangDash/MustangDash.ino:127-169 (bitmap + tri-bar
rectangles + ROM-font wordmark in one display list). Verified on the panel
2026-07-09: pony, tri-bar, and MUSTANG wordmark rendered as designed.
